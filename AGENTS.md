# Repository Guidelines

- **No binary files**: Only text-based assets may be committed. If you need fonts or other binary data, store them as Base64-encoded text files.
- **Testing**: After making changes, run `npm test` and capture the output. The project does not define a test script, so the command will fail; include this failure in your PR notes.
- **Formatting**: Keep code in JavaScript, HTML, or CSS tidy and readable. Use two spaces for indentation.
- **Pull Requests**: Summaries should describe major changes. Mention if tests fail or pass.
